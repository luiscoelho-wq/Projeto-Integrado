# Requisitos Não Funcionais (RNF)

RNF01 — Usabilidade
O sistema deve apresentar interface intuitiva, organizada e de fácil compreensão, permitindo uso sem necessidade de treinamento prévio.

RNF02 — Responsividade visual
A interface deve se adaptar adequadamente a diferentes tamanhos de tela de dispositivos móveis compatíveis.

RNF03 — Desempenho
As transições entre telas devem ocorrer de forma fluida, sem atrasos perceptíveis ao usuário.

RNF04 — Organização do código
O projeto deve seguir boas práticas de estruturação em Flutter, separando interface e lógica para facilitar manutenção.

RNF05 — Manutenibilidade
O código-fonte deve ser organizado em pastas e componentes reutilizáveis, permitindo futuras expansões.

RNF06 — Portabilidade
O sistema deve ser desenvolvido utilizando Flutter, possibilitando futura adaptação para múltiplas plataformas móveis.

RNF07 — Segurança acadêmica
O sistema deve utilizar apenas dados fictícios, não permitindo inserção de dados reais sensíveis.

RNF08 — Padronização visual
As telas devem manter consistência de cores, tipografia e componentes visuais conforme definido no protótipo.

RNF09 — Documentação
O projeto deve possuir documentação organizada no repositório GitHub, incluindo visão do produto e requisitos.

RNF10 — Versionamento
O desenvolvimento deve ser controlado por versionamento utilizando Git e GitHub.

RNF11 — Confiabilidade básica
O aplicativo não deve apresentar falhas críticas durante navegação entre as telas principais.

RNF12 — Clareza de escopo
As funcionalidades implementadas devem respeitar os limites definidos no escopo acadêmico do projeto.
